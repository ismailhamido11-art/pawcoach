Dans `computeHealthScore` (ligne ~329), la section "Vet visit score (0-25)" ne considere que les visites passees. Modifier pour aussi tenir compte de `dog.next_vet_appointment` :

**Logique actuelle :**
```js
const vetVisits = recs.filter(r => r.type === "vet_visit" && isValidDate(r.date));
let vetScore = 0;
if (vetVisits.length > 0) {
  const lastVet = vetVisits.sort((a, b) => new Date(b.date) - new Date(a.date))[0];
  const months = monthsAgo(parseDate(lastVet.date));
  if (months <= 6) vetScore = 25;
  else if (months <= 12) vetScore = 20;
  else if (months <= 18) vetScore = 10;
  else vetScore = 5;
}
```

**Logique modifiee :**
Apres le calcul du vetScore base sur les visites passees, ajouter un bonus pour le prochain RDV programme :

```js
// Bonus next_vet_appointment : un RDV programme dans les 30 jours montre une demarche proactive
if (dog?.next_vet_appointment && isValidDate(dog.next_vet_appointment)) {
  const apptDate = parseDate(dog.next_vet_appointment);
  if (apptDate) {
    const daysUntil = daysBetween(t, apptDate); // t = today()
    if (daysUntil >= 0 && daysUntil <= 30) {
      // RDV dans les 30j : si pas de visite recente, ce RDV vaut 15 points (pas 25 — pas encore fait)
      // Si visite recente + RDV -> plafonner a 25 (pas de double boost)
      vetScore = Math.min(25, vetScore + 15);
    }
  }
}
```

NOTE : `daysBetween(a, b)` calcule `b - a` en jours. Verifier dans healthStatus.js que `daysBetween(t, apptDate)` est bien positif pour une date future. Si la fonction fait `Math.floor((b-a)/ms)`, alors `daysBetween(today, futurDate)` est positif. Lire la definition de `daysBetween` dans le fichier avant d'ecrire le code.

Ne pas modifier la signature de `computeHealthScore`. Ne pas modifier les autres sections (vaccine, weight, activity). Ne pas toucher les 11 consumers existants.
